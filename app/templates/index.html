API Scheme

Contact API has support for Contacts with (O..N) numbers of emails, phone numbers and addresses. To simplify the Client side it was considered that contact can have:
- A personal email.
- A personal phone.
- A work email.
- A work phone
- One address.

API is considering Contacts from Brazil so phone numbers and Address should use brazilian patterns for those fields.
